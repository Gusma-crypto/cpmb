# Development Rules

## General Rules
- Use Laravel 12
- Use Blade + Tailwind
- Use Service Layer
- Controllers must stay thin
- Business logic inside Services
- Validation using FormRequest
- Use modular architecture

## Naming Rules
- Controller suffix: Controller
- Service suffix: Service
- Request suffix: Request

## Route Rules
- Use route prefix per module
- Use route names

## Database Rules
- Use snake_case
- Use foreign keys
- Use soft delete when needed

## UI Rules
- Clean admin dashboard
- Responsive layout
- Use sidebar navigation
